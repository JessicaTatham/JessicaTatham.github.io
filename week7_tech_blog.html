SQL vs. NoSQL

SQL databases

So far, while studying SQL this week, I've found it to be easy and user friendly. Here are a few key features about SQL:

  • Relational Database
  • Table based: SQL stores its data in tables that consist of rows and columns of data.
  • Vertically scalable
  • SQL databases use stuctured query language to define and manipulate data.
  • SQL is best used for complex query iintensice enviroments.

NoSQL databases

I have not yet used a NoSQL database, but from my research, I have found that it is extremely different from SQL. Here are some key fetures about NoSQL:

  • Non-Relational Database
  • Document based, key-value pairs, graph databases, or wide column stores: This allows NoSQL databases to not have to adhere to standard schema definitions
  • Horizontally scalable
  • NoSQL focuses on the collection of documents.
  • NoSQL is not a good fit for complex queries, but rather for large traffic horizonally scalable databases.

Both SQL and NoSQL have their places within the database world, it's just a matter of choosing which one will suit your purposes.
